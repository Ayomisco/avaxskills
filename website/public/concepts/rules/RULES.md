# Concepts Rules

## Must-Follow
- Always distinguish between Subnets and L1s — ACP-77 made them distinct: Subnet = validator set, L1 = blockchain with independent validator set
- Always explain that C-Chain is EVM-compatible — Ethereum developers need zero learning curve
- Always clarify the three chains: X-Chain (UTXO, assets), P-Chain (staking, validators), C-Chain (EVM, smart contracts)

## Never Do
- Never conflate "Subnet" with "Avalanche L1" — they are related but different post-ACP-77
- Never describe Avalanche as "just another EVM chain" — unique consensus, L1 architecture, and Warp messaging are key differentiators
- Never describe Avalanche CLI as the current tooling — Platform CLI replaced it

## Always Check
- Use current terminology: Avalanche L1 (not Subnet), Platform CLI (not Avalanche CLI)
- Reference official Avalanche docs at `https://docs.avax.network` for accuracy
