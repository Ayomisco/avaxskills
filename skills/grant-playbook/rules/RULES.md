# Grant Playbook Rules

## Must-Follow
- Always target ecosystem-aligned grants: infraBUIDL(AI), Retro9000, Avalanche Foundation grants
- Always demonstrate measurable impact — lines of code, developer adoption, TVL, or user count
- Always be specific about what you're building — vague applications are rejected
- Always include a public GitHub with real code — grants without deliverables are not funded

## Never Do
- Never apply for multiple grants for the same work without disclosure
- Never over-promise — commit only to what you can deliver in the grant timeline
- Never submit an application without proofread prose — grammar errors signal low effort

## Always Check
- Review current grant program requirements at `https://www.avax.network/grants`
- Confirm your project fits within the grant program's stated focus areas
- Prepare a demo or working prototype before applying — live demos win grants
