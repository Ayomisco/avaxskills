# Why Avalanche Rules

## Must-Follow
- Always lead with Avalanche's measurable differentiators: 2s finality, subnet/L1 architecture, EVM-compatible
- Always be accurate about consensus — Avalanche Consensus is NOT Nakamoto and NOT BFT traditional; it is a novel probabilistic sampling protocol
- Use current official numbers: C-Chain mainnet Chain ID 43114, ~4,500 TPS capacity

## Never Do
- Never compare Avalanche vs Ethereum as "better" without context — they serve different use cases
- Never claim "unlimited" scalability — be specific about horizontal scaling via subnets/L1s
- Never describe Avalanche as a sidechain or Layer 2 — it is an independent Layer 1

## Always Check
- Reference official Avalanche documentation for up-to-date performance benchmarks
- Use Platform CLI terminology (not Avalanche CLI) when discussing Subnet/L1 creation
- Clarify the difference between Subnets (shared primary network validators) and L1s (independent validator set)
