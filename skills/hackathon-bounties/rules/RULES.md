# Hackathon Bounties Rules

## Must-Follow
- Always read the full bounty requirements before starting — scope is enforced at judging
- Always deploy to the specified network (usually Fuji) — mainnet deployments don't count for testnet bounties
- Always include a working demo with public URL or video — judges don't run local code
- Submit before the deadline — most hackathons cut off exactly at deadline with no grace period

## Never Do
- Never submit a copy of a tutorial or example repo — plagiarism is disqualifying
- Never skip the README — judges read it first; if it doesn't explain your project clearly, you lose
- Never use deprecated Avalanche tooling in a hackathon submission — it signals outdated knowledge

## Always Check
- Confirm your project addresses the specific bounty judging criteria
- Test your demo flow end-to-end as a first-time user would experience it
- Include deployed contract addresses on the correct testnet in your submission
